도구

AI를 실제 개발과 업무에 활용하게 해주는 도구와 표준입니다. MCP, Claude Code, LangChain, 스킬·훅처럼 모델과 외부 세계를 잇는 기술들을 모았습니다.
  • 가드 스킬 Guard SkillsAI가 생성한 코드·테스트·문서가 흔히 빠지는 실패 패턴을, 작업이 끝난 변경분에 한 번 더 돌려 잡아내는 2차 품질 검증용 스킬.
  • 구스 Goose (Block AI Agent)Block이 만든 오픈소스 AI 에이전트로, 로컬 머신에서 실행되며 어떤 LLM이든 연결해 코드 작성부터 디버깅, 배포까지 자율적으로 수행한다.
  • 그래파이 Graphify코드베이스 전체를 질의 가능한 지식 그래프로 만들어 AI 코딩 어시스턴트가 파일을 일일이 읽지 않고도 구조를 파악하게 해주는 오픈소스 스킬.
  • 깃허브 코파일럿 GitHub CopilotGitHub과 OpenAI가 만든 AI 코딩 보조 도구. 코드 작성 중 다음 줄을 예측해 제안하는 방식으로 개발 속도를 높인다.
  • 다이나믹 워크플로우 Dynamic WorkflowsClaude가 작업을 받으면 그 자리에서 직접 오케스트레이션 스크립트를 짜서 수십~수백 개의 서브에이전트를 병렬로 돌리고 결과를 검증해 하나의 답으로 정리하는 Claude Code 기능.
  • 데빈 DevinCognition이 만든 최초의 AI 소프트웨어 엔지니어. 브라우저, 에디터, 터미널을 스스로 조작하며 코딩 작업을 자율적으로 수행한다.
  • 도구 사용 Tool Use / Function CallingAI 모델이 텍스트 생성을 넘어 외부 함수를 직접 호출해 실제 작업을 수행하게 하는 메커니즘. 에이전트가 "손"을 가질 수 있게 해주는 핵심 기능이다.
  • 라마인덱스 LlamaIndex다양한 데이터 소스를 LLM에 연결하기 위한 프레임워크. RAG 파이프라인 구축에 특화되어 있다.
  • 라우팅 패턴 Routing Pattern요청의 복잡도와 특성에 따라 가장 적합한 LLM 모델을 자동으로 선택해 전달하는 시스템.
  • 랭스미스 LangSmithLangChain이 만든 LLM 앱 디버깅·테스트·모니터링 플랫폼. LangChain 파이프라인과 자연스럽게 연동된다.
  • 랭체인 LangChainLLM 호출, 메모리 관리, 도구 연동, 에이전트 오케스트레이션을 하나의 인터페이스로 묶어주는 오픈소스 프레임워크. Python AI 에이전트 개발의 사실상 표준이다.
  • 랭퓨즈 LangfuseLLM 애플리케이션의 실행 흐름을 추적하고 품질을 측정하는 오픈소스 관측성 플랫폼.
  • 모션 애니씽 motion-anything (Agentic Motion Engine)채팅으로 원하는 느낌을 묘사하면 AI가 직접 애니메이션을 만들어주는 오픈소스 에이전틱 모션 엔진. 8개 코딩 에이전트와 403개 모션 레시피를 기반으로 JSON·CSS·React·Lottie·MP4·GIF 등 다양한 형식으로 출력한다.
  • 바오위 디자인 Baoyu Design코딩 에이전트가 UI 목업·프로토타입·슬라이드 덱을 브라우저에서 바로 열리는 자체 완결형 HTML로 만들어내게 해주는 에이전트 스킬.
  • 슈퍼파워스 SuperpowersClaude Code가 코드를 막 짜기 전에 브레인스토밍·계획·테스트·리뷰 같은 개발 규율을 강제로 거치게 만드는 오픈소스 스킬 묶음(플러그인)이다. AI를 '빠른 타이피스트'에서 '규율 있는 개발팀'으로 바꾸는 게 목적이다.
  • 스킬 Skills / Slash CommandsClaude Code에서 슬래시(/)로 호출하는 재사용 가능한 자동화 워크플로우. 마크다운으로 정의하면 복잡한 작업을 명령 한 줄로 실행할 수 있다.
  • 앤트로픽 SDK Anthropic SDKClaude API를 Python과 TypeScript에서 편리하게 쓸 수 있도록 Anthropic이 제공하는 공식 라이브러리.
  • 에이전트 CLI Agent-Friendly CLIAI 에이전트가 GUI 대신 CLI를 선호하는 현상, 그리고 에이전트가 쓰기 좋게 CLI를 재설계하는 움직임. 구조화된 출력과 조합성이 에이전트 시대에 CLI를 다시 부활시켰다.
  • 오픈 메모리 프로토콜 Open Memory ProtocolClaude, ChatGPT, Cursor 같은 서로 다른 AI 도구들이 하나의 메모리 저장소를 공유할 수 있게 해주는 오픈 프로토콜. 매번 새 도구에서 처음부터 설명하지 않아도 이전 대화 맥락이 이어진다.
  • 오픈 웹UI Open WebUI로컬 LLM을 ChatGPT처럼 쓸 수 있게 해주는 오픈소스 웹 인터페이스. Ollama와 함께 로컬 AI 스택의 사실상 표준 프론트엔드로 자리잡았다.
  • 오픈AI SDK OpenAI SDKOpenAI의 GPT 시리즈와 기타 모델을 Python, TypeScript 등에서 쓸 수 있는 공식 라이브러리.
  • 올라마 Ollama로컬에서 LLM을 한 줄 명령으로 실행할 수 있게 해주는 도구. Docker가 컨테이너를 쉽게 만들듯, Ollama는 LLM을 쉽게 돌린다.
  • 위브 라우터 Weave Router에이전트 코딩 도구의 모든 요청을 비용 대비 최적의 모델로 자동 라우팅하는 프록시로, 기존 예산으로 약 2배 더 많은 작업을 처리할 수 있다.
  • 윈드서프 WindsurfCodeium이 만든 AI 코드 편집기. 에이전트 기반의 Cascade 기능으로 여러 파일에 걸친 복잡한 작업을 자동 처리한다.
  • 커서 CursorAI 기능이 깊이 내장된 코드 편집기. VS Code 기반으로 자연어 지시로 코드를 작성·수정·리팩토링할 수 있다.
  • 컨텍스트 캐싱 Context Caching긴 시스템 프롬프트나 문서를 매 요청마다 재처리하지 않고 캐시해서 비용과 지연 시간을 줄이는 기법.
  • 케이브맨 CavemanAI가 군더더기 없이 원시인처럼 짧게 말하게 만들어 출력 토큰을 크게 줄이는 Claude Code 스킬.
  • 코덱스 OpenAI CodexOpenAI가 만든 클라우드 기반 AI 코딩 에이전트. 샌드박스 환경에서 코드를 읽고, 쓰고, 실행하며 GitHub 이슈와 PR을 자율적으로 처리한다.
  • 코드번 CodeburnClaude Code나 Codex 같은 AI 코딩 도구가 토큰을 어디에 얼마나 썼는지 터미널 대시보드로 한눈에 보여주는 오픈소스 도구.
  • 코파일럿 SDK Copilot SDKGitHub Copilot의 AI 기능을 외부 앱이나 IDE에 통합할 수 있게 해주는 공식 SDK. Copilot의 코드 완성과 채팅 기능을 자체 도구에 임베드할 수 있다.
  • 클로드 디스패치 Claude DispatchClaude가 사용자의 요청을 분석해 여러 전문 에이전트에게 자동으로 분배하고 결과를 종합하는 Anthropic의 멀티에이전트 기능.
  • 클로드 코드 Claude Code CLI터미널에서 자연어로 코드베이스를 이해하고, 파일을 편집하고, Git 워크플로우 전체를 자율적으로 수행하는 AI 코딩 에이전트.
  • 트레이싱 LLM TracingLLM 앱의 각 실행 단계를 기록해서 어떤 일이 일어났는지 추적하는 것. 복잡한 에이전트 파이프라인 디버깅에 필수다.
  • 프롬프트 관리 Prompt Management프롬프트를 코드처럼 버전 관리하고, 팀이 협업하며, 성능 변화를 추적하는 체계적인 접근 방식.
  • 플레이그라운드 AI Playground코드 없이 브라우저에서 바로 LLM을 호출하고 프롬프트를 테스트할 수 있는 웹 인터페이스.
  • 훅 HooksClaude Code에서 특정 이벤트 전후에 자동으로 실행되는 셸 명령. 프롬프트가 아니라 운영체제가 직접 실행하므로 100% 확실하게 동작한다.
  • AGENTS.mdAI 에이전트가 프로젝트를 이해하고 올바르게 행동하도록 가이드하는 마크다운 문서. CLAUDE.md의 멀티에이전트 버전이다.
  • AI IDE AI-Powered IDEAI 코드 보조 기능이 깊이 통합된 개발 환경. 단순 자동완성을 넘어 코드 생성, 리팩토링, 디버깅을 AI와 함께 한다.
  • auth.md auth.md Agent Authentication ProtocolAI 에이전트가 외부 서비스에 접근할 때 필요한 인증 방법을 명시하는 마크다운 파일 협약. CLAUDE.md가 에이전트의 행동 방식을 안내하듯, auth.md는 에이전트가 API 키·OAuth·권한 범위 같은 인증 정보를 어떻게 처리해야 하는지를 사람과 에이전트 모두가 읽을 수 있는 형태로 정의한다.
  • CLAUDE.mdClaude Code가 세션 시작 시 자동으로 읽는 프로젝트 설정 파일. 코드만으로는 알 수 없는 규칙 — 빌드 명령, 코딩 컨벤션, 아키텍처 결정 — 을 담는다.
  • DSPy프롬프트를 직접 작성하는 대신 프로그래밍 방식으로 LLM 파이프라인을 최적화하는 프레임워크.
  • LiteLLM100개 이상의 LLM을 단일 OpenAI 호환 인터페이스로 호출할 수 있는 오픈소스 라이브러리.
  • LLM 게이트웨이 LLM Gateway여러 LLM 제공자를 하나의 통합 API로 묶어주는 중간 레이어. 모델을 바꿔도 코드를 수정하지 않아도 된다.
  • MCP Model Context ProtocolAI 모델이 외부 도구와 데이터에 접근하기 위한 표준 프로토콜. USB-C가 기기를 하나의 규격으로 연결하듯, MCP는 AI와 외부 시스템을 하나의 방식으로 연결한다.
  • n8n n8n (nodemation)코드 없이 AI 에이전트와 자동화 워크플로우를 시각적으로 구축할 수 있는 오픈소스 플랫폼. 400개 이상의 통합과 LangChain 기반 AI 노드를 제공하며, 셀프호스팅으로 데이터 통제권을 유지할 수 있다.
